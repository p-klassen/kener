# Admin UI i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add per-user language preference to the Kener admin interface so each admin can choose their own display language, independent of the public status page locale.

**Architecture:** A new `preferred_locale` column on `users` stores the choice. The manage `+layout.ts` (new file) writes this value to `localStorage` before calling `i18n.init()`, so the existing store picks it up. A language picker in the Account dialog in the sidebar saves back to DB via a new `updateUserLocale` API action. All admin UI strings are then migrated to `$t("manage....")` keys in `en.json`.

**Tech Stack:** SvelteKit 5 (Svelte 5 runes), TypeScript, Knex.js (SQLite/PG/MySQL), shadcn-svelte `<Select>`, existing `src/lib/stores/i18n.ts` (`i18n.init`, `i18n.setLocale`).

## Task 5: Create manage `+layout.ts`

This new client-side load function writes the DB locale preference to localStorage before `i18n.init()` reads it, so the DB value wins over any stale browser value.

**Files:**
- Create: `src/routes/(manage)/+layout.ts`

- [ ] **Step 1: Create the file**

```typescript
import { i18n } from "$lib/stores/i18n";
import type { LayoutLoad } from "./$types";

export const load: LayoutLoad = async ({ data, fetch }) => {
  const { userLocale, availableLocales, defaultLocale } = data;

  if (userLocale && typeof window !== "undefined") {
    localStorage.setItem("kener_preferred_locale", userLocale);
  }

  await i18n.init(availableLocales, defaultLocale ?? "en", fetch);
  return {};
};
```

- [ ] **Step 3: Add `saveLocale` function after the `openAccountDialog` function**

```typescript
async function saveLocale(locale: string) {
  savingLocale = true;
  localeError = "";
  try {
    const response = await fetch(clientResolver(resolve, "/manage/api"), {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ action: "updateUserLocale", data: { locale } }),
    });
    const resp = await response.json();
    if (resp.error) {
      localeError = resp.error;
    } else {
      currentLocale = locale;
      await i18n.setLocale(locale);
    }
  } catch {
    localeError = "Error saving language preference";
  } finally {
    savingLocale = false;
  }
}
```
